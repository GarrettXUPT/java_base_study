网络通信协议：
    TCP/IP协议：传输控制协议/因特网互联协议
    网络协议分类：
        TCP协议：面向连接的通信协议，在数据传输之前要先建立连接，再进行通信。提供了计算机之间可靠的
                误差错的数据传输，在TCP协议中，要明确客户端和服务器，每次创建连接都要进行三次握手。
              三次握手：
                第一次握手：客户端向服务器端发送建立连接的请求，等待服务器确认
                第二次握手：服务器端向客户端回送一个响应，表明请求已经收到了
                第三次握手：客户端再次向服务器端发送确认信息，确认连接

        UDP协议：用户数据报协议，面向无连接的协议，在数据传输时，数据的发送端和接收端不需要建立连接
                不能保证数据的完整性，但是传输较快，但是不可以使用UDP传输重要的数据

网络编程要素：
    协议：
        计算机通信之间的规则
    IP地址：
        互联网协议地址，IP地址是计算机在网络中唯一的编号。若电脑是个电话的话，IP地址就是一个电话号码
        IPV4：是一个32位二进制数，通常被分为4个字节
        IPV6：采用128位地址的长度，每16字节一组，分成8组十六进制数
    端口号：
        用两个字节表示的整数
        计算机通过网络可以连接在一起，那么计算机之间就可以传输数据进行通信，我们可以通过IP地址找到对方的计算机
        注意：
            我们一定要保证数据可以准确无误的发送到对方计算机的软件上
        端口号是一个逻辑端口，我们无法直接看到，但是我们可以借助一些软件进行查看，当我们使用网络软件时，那么操作系统就会
        为网络软件分配一个随机的端口号或者是打开时，软件会找操作系统要一个指定的端口号
        注意：
            1024之前的端口号是不可以使用的，因为已经被分配出去了，网络的端口号是不可以重复的
        我们把IP地址加上端口号，就可可以准确无误的将数据发送至对方计算机的指定软件中
        常用端口：
            1、80端口：网路端口
            2、数据库端口：mysql 3306 Oracle 1521
            3、TomCat服务器：8080

TCP通信程序：
    TCP可以实现两台计算机之间的数据交互，通信的两端，需要严格区分为客户端与服务端
    两端通信的步骤：
        1、服务端程序需要事前启动，等待客户端的连接
        2、客户端主动连接服务器端，连接成功以后才可以进行通信，服务端不可以主动连接
        3、客户端和服务端之间的连接包含一个(IO)对象，客户端就可以使用IO对象进行通信，通信的数据不仅仅是字符，所以IO对象是字节流对象

    服务器端：配置高的计算机 ip端口号 SeverSocket类
        2、服务器接收来自客户端的数据
            InputStream: 你好服务器
        3、服务器发送数据到客户端
            OutputStream: 收到了 谢谢
        在服务器端需要注意事项：
            1、多个客户端和服务器端进行通信，服务器端必须明确和哪个客户端进行的交互
               在服务器端存在一个方法accept客户端获取到请求客户端对象
            2、多个客户端和服务器进行交互，就可以使用多个IO流对象
               服务器没有IO流，但是服务器可以获取到请求的客户端Socket，使用每个客户端Socket中提供的IO流和客户端进行交互
               服务器使用客户端的字节输入流读取客户端发送的数据，服务器使用客户端的字节输出流，给客户端发送数据
               服务器使用客户端的流和客户端进行交互

    客户端：配置较低的笔记本 ip端口号 Socket类
        1、客户端发送数据给服务器端
           OutputStream: 你好服务器
        4、客户端接收来自服务器的数据
            InputStream: 收到了 谢谢

    TCP通信的客户端，向服务器发送连接请求，给服务器发送数据，读取服务器回写的数据
    java.net.Socket 此类实现的是客户端套接字，套接字是两台机器间通信的端点
        套接字：包含了IP地址和端口的网络单位
        构造方法：
             Socket(String host, int port) 创建一个流套接字并将其连接在主机上指定的端口号
             参数：
                String host:服务器主机的名称/服务器的IP地址
                int port: 服务器的端口号
        成员方法：
             OutputStream getOutputStream() 返回套接字的输出流
             InputStream getInputStream() 返回套接字的输入流
             void close() 关闭套接字
        实现步骤：
             1、创建一个客户端对象Socket，构造方法绑定服务器的IP地址和端口号
             2、使用Socket对象中的方法getOutputStream()获取网络字节输出流对象OutputStream
             3、使用网络字节输出流对象OutputStream方法write向服务器发送数据
             4、使用Socket对象中的方法getInputStream()获取网络字节输入流对象InputStream
             5、网络字节输入流对象InputStream的方法read读取服务器回写的数据
             6、释放资源
           注意事项：
                1、客户端与服务器进行的交互必须使用Socket中提供的网络流，不能使用自己创建的流对象
                2、我们创建客户端Socket对象的时候，就会去请求服务器和服务器进行三次握手建立连接通路
                   这时服务器还没有启动那么就会抛出异常，若服务器已经启动，那么就可以进行交互

    TCP通信的服务器端：接收客户端的请求，读取客户端发送的数据，给客户端回写数据
        表示服务器的类 java.net.ServerSocket
        构造方法：
            ServerSocket(int port) 创建绑定到特定端口的套接字
        必须明确的事：必须知道哪个用户端请求的服务器，所以可以使用accept方法获取到客户端的Socket
        成员方法：
            Socket accept() 监听并接收到该套接字的连接
        实现步骤：
            1、创建ServerSocket对象，构造方法中传递指定的端口号
            2、使用ServerSocket对象中的方法accept，获取到请求的客户端对象Socket
            3、使用网络字节输出流对象OutputStream方法write向服务器发送数据
            4、使用网络字节输出流InputStream对象中的方法read，读取客户端发送的数据
            5、使用Socket对象中的方法getOutputStream()获取网络字节输出流对象OutputStream
            6、使用网络字节输出流对象OutputStream对象中的方法write，给客户端回写数据
            7、释放资源
